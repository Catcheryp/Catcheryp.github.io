0x03 绕过

绕过主要看这一段代码:

  • 变量 $data 弱等于字符串 bugku is a nice plateform!
  • 变量 $id 弱等于整型数 0
  • 变量 $b 的长度大于 5
  • 字符串 1114 要与字符串 111 连接变量 $b 的第一个字符构成的正则表达式匹配
  • 变量 $b 的第一个字符弱不等于整型数 4

1.PHP 弱类型比较

$id==0

变量 $id 若想满足非空非零且弱等于整型数 0，则 $id 的值只能为非空非零字符串，这里假设 $id = "asd"

2.PHP 伪协议

$data = @file_get_contents($a,'r');

$data=="bugku is a nice plateform!"

源码中变量 $data 是由 file_get_contents() 函数读取变量 $a 的值而得，所以 $a的值必须为数据流。

这里用伪协议 php://来访问输入输出的数据流，其中 php://input 可以访问原始请求数据中的只读流。这里令 $a = "php://input"，并在请求主体中提交字符串 bugku is a nice plateform!

3.eregi() 截断漏洞

strlen($b)>5

eregi("111".substr($b,0,1),"1114")

substr($b,0,1)!=4

ereg() 函数或 eregi() 函数存在空字符截断漏洞，即参数中的正则表达式或待匹配字符串遇到空字符则截断丢弃后面的数据。

源码中待匹配字符串（第二个参数）已确定为 "1114"，正则表达式（第一个参数）由 "111" 连接 $b 的第一个字符组成，若令 substr($b,0,1) = "\x00"，即满足 "1114" 与 "111" 匹配。因此，这里假设 $b = "\x0012345"，才能满足以上三个条件。

在构造变量 b 中的空字符时，过早将空字符 \x00 放入，在提交请求时导致请求头截断，继而请求失败，得不到响应，空字符应该在后台脚本的变量中出现，而不是在 URL 查询字符串变量中出现。
